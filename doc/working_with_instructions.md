
1) arm-none-eabi-objdump -d is your friend, use it to dump the encodings from reference implementations.
2) instruction-test-bench can be utilized to synthezise instructions from reliable source (gcc)
3) build.rs contains the autogenerated decoding -- follow from there to the actual decoder funcs
4) when adding new instruction support, always add a test(s). Decoding is one thing, but execution also needs testing.
